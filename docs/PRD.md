# Product Requirements Document (PRD)
# Academic Web Portfolio

**Project Name:** Academic Web Portfolio  
**Project Type:** Personal Portfolio Website  
**Version:** 2.0  
**Status:** In Development

---

# 1. Introduction

## 1.1 Purpose

The Academic Web Portfolio is a personal website developed as part of the On-the-Job Training (OJT) requirements for the Bachelor of Science in Information Technology program at Mapúa Malayan Colleges Laguna.

The website serves as a centralized platform to showcase the portfolio owner's academic journey, technical skills, software development projects, certificates, seminars, and capstone project throughout college.

The portfolio is designed to be professional, responsive, and easy to navigate while highlighting the owner's growth as an aspiring software developer.

---

## 1.2 Project Goals

The website aims to:

- Present the portfolio owner's college journey in a clear and organized manner.
- Showcase selected software projects developed from the first to fourth year.
- Highlight technical skills acquired throughout college.
- Present relevant certificates and seminars.
- Feature the Capstone Project as the primary academic achievement.
- Allow visitors to download the Curriculum Vitae.
- Serve as a professional portfolio for both academic evaluation and future career opportunities.

---

# 2. Project Scope

## Included

The website shall include the following sections:

- Hero
- About Me
- Technical Skills
- College Journey
- Software Developed
- Capstone Project
- Certificates
- Seminars
- Contact
- Footer

Additional functionality includes:

- Download Curriculum Vitae (PDF)
- Responsive Design
- Smooth Scrolling Navigation

---

## Excluded

The following features are outside the scope of the project:

- User Authentication
- Database Integration
- Backend Development
- Content Management System (CMS)
- Blog Functionality
- Online Messaging
- User Accounts

---

# 3. Target Audience

The primary users of the website include:

- OJT Adviser
- OJT Coordinator
- Faculty Members

- Recruiters
- Future Employers

# 5. Website Structure

The portfolio follows a single-page scrolling layout.

The recommended section order is:

1. Hero
2. About Me
3. Technical Skills
4. College Journey
5. Software Developed
6. Capstone Project
7. Certificates
8. Seminars
9. Contact
10. Footer

---

# 6. Functional Requirements

## 6.1 Hero

The Hero section shall display:

- Portfolio Owner's Name
- Degree Program
- School
- Short Professional Introduction
- Download CV Button

---

## 6.2 About Me

The About section shall provide a short introduction describing the portfolio owner's academic background, interests, and career aspirations.

---

## 6.3 Technical Skills

The Technical Skills section shall categorize the owner's skills into:

- Programming Languages
- Frameworks & Libraries
- Development Tools
- Database Technologies
- Version Control

Skills shall be presented using badges or cards.

---

## 6.4 College Journey

The College Journey section shall present the owner's academic progression from the first year to the fourth year using a timeline or milestone layout.

Each year may include:

- Major subjects
- Significant projects
- Important achievements

---

## 6.5 Software Developed

The Software Developed section shall showcase software projects completed throughout college.

Each project shall include:

- Project Name
- Academic Year
- Short Description
- Discussion of Features
- GUI Screenshots
- Software Used

Projects should be grouped according to academic year.

---

## 6.6 Capstone Project

The Capstone Project section shall include:

- Project Title
- Project Description
- Discussion of Features
- GUI Screenshots
- Software Used
- Hardware Used

---

## 6.7 Certificates

The Certificates section shall display academic, technical, and professional certificates earned by the portfolio owner.

Each certificate may include:

- Certificate Title
- Issuing Organization
- Date Received
- Certificate Image

---

## 6.8 Seminars

The Seminars section shall list seminars attended during college.

Each seminar may include:

- Seminar Title
- Organizer
- Date
- Short Description
- Certificate (if available)

---

## 6.9 Contact

The Contact section shall provide:

- Email Address
- GitHub
- LinkedIn

Additional contact methods may be added in future versions.

---

# 7. Non-Functional Requirements

The website shall be:

- Responsive across desktop, tablet, and mobile devices.
- Easy to navigate.
- Fast loading.
- Accessible.
- Organized.
- Maintainable.
- Professional in appearance.
- Content-focused rather than text-heavy.
- Visually balanced through proper spacing, typography, and imagery.
- Suitable for both academic presentation and future professional use.

---

# 8. Responsive Design

The website shall support:

## Desktop

- Multi-column layouts where appropriate.
- Spacious layouts.
- Full navigation bar.

---

## Tablet

- Responsive grid layouts.
- Comfortable spacing.
- Optimized typography.

---

## Mobile

- Single-column layout.
- Collapsible navigation.
- Touch-friendly buttons.
- Readable text without horizontal scrolling.

---

# 9. Success Criteria

The project shall be considered successful when:

- All required portfolio sections are completed.
- Information is presented clearly and professionally.
- Content is concise and easy to scan.
- The website maintains a clean and consistent visual design.
- The website is fully responsive across all supported devices.
- The Curriculum Vitae can be downloaded successfully.
- Software projects are properly documented.
- Certificates and seminars are organized clearly.
- Navigation functions correctly.
- The website is successfully deployed online.

The project shall:

- Remain frontend-only.
- Use the approved technology stack.
- Maintain a simple and professional design.
- Avoid unnecessary complexity.
- Follow the UI Guidelines throughout development.
