# UI Guidelines
# Academic Web Portfolio

**Project Name:** Academic Web Portfolio  
**Project Type:** Personal Portfolio Website  
**Version:** 2.0

---

# 1. Purpose

This document defines the visual design system and user interface guidelines for the Academic Web Portfolio.

The objective is to create a clean, modern, and professional portfolio that showcases the portfolio owner's academic journey, technical skills, software projects, certificates, seminars, and capstone project.

The website should prioritize readability, consistency, responsiveness, and ease of navigation while maintaining a polished and engaging user experience.

---

# 2. Design Principles

The interface should follow these principles throughout the website.

- Clean and modern
- Professional
- Content-focused
- Minimalist
- Easy to navigate
- Consistent
- Responsive
- Accessible

The website should avoid unnecessary visual clutter and allow the portfolio content to remain the primary focus.

---

# 3. Design Inspiration

The overall layout is inspired by modern editorial portfolio websites, particularly the clean structure and spacious layouts of the Odoo Design website.

The portfolio should adopt similar design principles while maintaining its own unique identity.

Characteristics include:

- Large typography
- Spacious layouts
- Generous whitespace
- Alternating section layouts
- Rounded cards
- Clean grids
- Smooth scrolling
- Subtle hover interactions
- Clear visual hierarchy

The website should not copy another design directly but instead use these principles as inspiration.

---

# 4. Visual Identity

The Academic Web Portfolio should communicate:

- Professionalism
- Technical competence
- Creativity
- Simplicity
- Organization

The interface should feel modern and polished without becoming flashy or distracting.

Visual interest should be created through thoughtful use of typography, spacing, layout, color, and subtle motion.

---

# 5. Design Theme

The Academic Web Portfolio follows a modern editorial design inspired by clean portfolio and agency websites.

The overall aesthetic takes subtle inspiration from Gojo Satoru's color palette and atmosphere through cool blue tones, soft lighting, and minimal geometric elements.

This inspiration should remain understated and should never become the primary focus of the website.

The portfolio should be recognized as a professional academic and career portfolio rather than an anime-inspired website.

Examples include:

- Cool blue color palette
- Soft gradient backgrounds
- Subtle glow effects
- Rounded corners
- Minimal geometric patterns
- Spacious layouts
- Clean typography
- Gentle transitions

The design should communicate elegance, clarity, and modernity rather than anime culture.

---

## Decorative Elements

Decorative elements may include:

- Soft blue gradients
- Minimal geometric patterns
- Thin line accents
- Soft glow effects
- Subtle background textures

Decorative elements should always remain secondary to the portfolio content.

---

## Elements to Avoid

The website should **not** include:

- Anime character illustrations
- Manga panels
- Anime screenshots
- Character silhouettes
- Domain Expansion graphics
- Jujutsu Kaisen logos
- Fan art
- Decorative stickers
- Cartoon illustrations
- Clip art
- Obvious anime references

The goal is for visitors to see a professional portfolio first. Any inspiration from Gojo Satoru should only be recognizable through the overall atmosphere, color palette, and visual style.

---

# 6. Professional Presentation

The portfolio should maintain a professional and minimal appearance throughout the website.

Do not use:

- Emojis
- Decorative emoji icons
- Emoji section headings
- Emoji buttons
- Emoji badges
- Cartoon graphics
- Playful stickers
- Excessive decorative illustrations

The interface should rely on:

- Typography
- Layout
- White space
- Color palette
- Soft animations
- High-quality UI components

Professional presentation should always take priority over decorative elements.

# 9. Website Layout

The Academic Web Portfolio follows a single-page scrolling layout.

The recommended section order is:

1. Hero
2. About Me
3. Technical Skills
4. College Journey
5. Software Developed
6. Capstone Project
7. Certificates
8. Seminars
9. Contact
10. Footer

Each section should occupy sufficient vertical spacing to create a comfortable reading experience.

Content should breathe naturally through generous whitespace.

Avoid overcrowding sections with excessive text or visuals.

Sections should alternate between text-first and visual-first layouts where appropriate to create visual rhythm and prevent the page from feeling repetitive.

---

# 10. Navigation

The navigation bar should remain fixed at the top of the page while scrolling.

Navigation Items

- Home
- About
- Skills
- Journey
- Projects
- Capstone
- Certificates
- Seminars
- Contact

The **Download CV** button should appear as the primary call-to-action on the navigation bar.

### Navigation Guidelines

- Highlight the active section while scrolling.
- Collapse into a mobile navigation menu on smaller screens.
- Keep navigation labels concise.
- Use smooth scrolling between sections.
- Maintain consistent spacing between navigation items.
- Keep the navigation clean and uncluttered.

---

# 11. Hero Section

## Purpose

The Hero section should immediately introduce the portfolio owner and communicate the purpose of the website.

## Content

The Hero section should include:

- Full Name
- Degree Program
- School
- Short Professional Introduction
- Download CV Button

A personal profile picture should not be included.

Instead, the Hero section should establish the visual identity of the portfolio through strong typography, generous whitespace, and subtle background elements.

## Design Guidelines

The Hero section should rely on:

- Strong typography
- Generous whitespace
- Clear visual hierarchy
- Subtle background gradients
- Soft blue glow effects
- Minimal geometric patterns

The Hero should establish the visual identity of the portfolio without distracting from the content.

The design should feel calm, elegant, and modern.

---

# 12. Section Layout

Each section should follow a consistent layout structure.

Recommended structure:

- Small Section Label *(Optional)*
- Section Title
- Short Introduction *(Optional)*
- Main Content

### Layout Guidelines

- Maintain generous spacing between sections.
- Align content consistently.
- Avoid large blocks of uninterrupted text.
- Break information into readable sections.
- Use cards or grids where appropriate.
- Prioritize readability over decoration.

Alternate the arrangement of text and visuals where appropriate to create visual rhythm throughout the page.

---

# 13. Visual Effects

Subtle visual effects may be used throughout the website to enhance the user experience.

Recommended effects include:

- Soft radial gradients
- Background blur
- Glassmorphism (used sparingly)
- Thin geometric line patterns
- Abstract circular outlines
- Low-opacity gradient overlays
- Frosted surfaces
- Soft blue glow effects

Visual effects should never compete with the content.

They should reinforce the overall atmosphere while maintaining excellent readability.

---

Recommended hover effects include:

- Border color transition

- Soft glow

Used for:

Used for:

- Clear hover states

---

# 16. Images

Images include:

- GUI screenshots
- Certificate previews
- Project images

### Guidelines

- Maintain original aspect ratio.
- Avoid stretching or distortion.
- Use rounded corners.
- Keep image dimensions consistent.
- Optimize images before deployment.
- Display screenshots clearly without unnecessary overlays.

Images should support the content rather than dominate the page.

---

# 17. Icons

Use **Lucide React** as the primary icon library.

Icons should only be used when they improve usability or readability.

Recommended usage includes:

- Navigation
- Contact Information
- Technical Skills
- Buttons
- External Links

### Icon Guidelines

- Maintain a consistent icon size.
- Use icons sparingly.
- Pair icons with descriptive text when appropriate.
- Avoid decorative icon usage.

Do **not** use emojis as interface icons.

Examples of elements that should not appear:

- Emoji section headings
- Emoji buttons
- Emoji cards
- Emoji badges
- Decorative emoji graphics

The interface should rely on typography, spacing, color, and layout rather than emojis to communicate information.


# 18. Content Presentation

The Academic Web Portfolio should prioritize clarity and readability over lengthy descriptions.

Visitors should be able to quickly understand the portfolio owner's skills, projects, and achievements without reading large blocks of text.

## Writing Style

Content should be:

- Clear
- Concise
- Professional
- Easy to scan
- Direct

Avoid overly academic or overly technical language unless necessary.

---

## Paragraph Length

Whenever possible:

- Keep paragraphs between 2–4 sentences.
- Avoid long paragraphs.
- Break complex information into smaller sections.

---

## Use of Lists

Use bullet points whenever they improve readability.

Recommended for:

- Technical Skills
- Software Used
- Hardware Used
- Features
- Certificates
- Seminar Details
- Programming Languages

Avoid unnecessary paragraphs when a list communicates the same information more effectively.

---

## Project Descriptions

Each software project should briefly communicate:

- Purpose
- Main Features
- Technologies Used
- Key Learning

Descriptions should remain concise.

The project itself should become the primary focus through screenshots and visual presentation.

---

## Capstone Presentation

The Capstone Project should receive more emphasis than other projects.

Present information in the following order:

- Project Title
- Short Overview
- Key Features
- Technologies Used
- Hardware Used
- GUI Screenshots

Avoid long technical explanations.

---

## Certificates

Certificates should be displayed using cards or a clean grid layout.

Each certificate should include only:

- Title
- Issuing Organization
- Year

Additional details should only appear when necessary.

---

## Seminars

Seminars should be displayed using concise cards.

Each seminar should include:

- Title
- Organizer
- Date

Descriptions should remain short.

---

## Technical Skills

Avoid listing every technology equally.

Instead, organize skills into categories.

Example:

Programming Languages

Frameworks

Databases

Development Tools

Version Control

Present skills using clean badges or chips rather than paragraphs.

---

## College Journey

The College Journey should highlight milestones rather than every academic event.

Each academic year should summarize:

- Major subjects completed
- Significant software projects
- Important achievements
- Skills gained

Keep each year's content brief and visually balanced.

---

## Visual Balance

Every section should maintain a balance between:

- Text
- Images
- White space

Avoid sections that are dominated by text.

Images, cards, and spacing should help guide the reader naturally through the portfolio.

Cards should present information clearly without excessive text.

---

## Overall Reading Experience

Visitors should be able to understand the portfolio within a few minutes.

The website should encourage scanning rather than extensive reading.

Content should remain concise, with visuals supporting the information rather than overwhelming it.

The overall experience should feel clean, modern, and professional, making the portfolio suitable for both academic evaluation and future career opportunities.
