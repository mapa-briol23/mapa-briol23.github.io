# UI Agent Guide
# Academic Web Portfolio

This document provides instructions for AI assistants responsible for designing and implementing the user interface of the Academic Web Portfolio.

This document should always be used together with:

- PRD.md
- UI_GUIDELINES.md
- CLAUDE.md

---

# 1. Purpose

The UI agent is responsible for implementing a clean, modern, and professional interface that follows the project's design requirements.

The UI should prioritize clarity, readability, consistency, and responsiveness while maintaining a polished user experience.

---

# 2. Responsibilities

The UI agent should:

- Follow the requirements defined in PRD.md.
- Follow the design system defined in UI_GUIDELINES.md.
- Follow the implementation guidelines defined in CLAUDE.md.
- Build reusable UI components.
- Maintain visual consistency throughout the website.
- Ensure responsive layouts across supported devices.

The UI agent should not introduce new features or significantly alter the approved design without justification.

---

# 3. Design Priorities

When making UI decisions, prioritize the following:

1. Readability
2. Simplicity
3. Consistency
4. Responsiveness
5. Accessibility
6. Performance
7. Visual polish

If multiple design solutions are possible, choose the simplest solution that satisfies the project requirements.

---

# 4. Content Presentation

The UI should encourage visitors to scan information rather than read lengthy text.

Whenever appropriate:

- Keep sections concise.
- Use cards for grouped information.
- Use grids where appropriate.
- Maintain generous whitespace.
- Present information with a clear visual hierarchy.

Avoid creating layouts dominated by large blocks of text.

---

# 5. Components

The UI agent should reuse existing components whenever possible.

Examples include:

- Button
- Card
- Section Header
- Project Card
- Certificate Card
- Seminar Card
- Skill Badge

Avoid creating duplicate components with similar functionality.

---

# 6. Responsive Design

Every interface should work across:

- Mobile
- Tablet
- Desktop

Layouts should adapt naturally without sacrificing readability or usability.

---

# 7. Visual Consistency

Maintain consistency in:

- Colors
- Typography
- Spacing
- Border radius
- Shadows
- Icons
- Buttons
- Cards

Do not introduce new visual styles that conflict with the established design.

---

# 8. Accessibility

The UI should:

- Use semantic HTML.
- Maintain sufficient color contrast.
- Support keyboard navigation where applicable.
- Include descriptive alt text for images.
- Preserve readable typography across devices.

---

# 9. Things to Avoid

Avoid:

- Overly complex layouts
- Excessive animations
- Large blocks of text
- Inconsistent spacing
- Unnecessary decorative elements
- Emojis in the interface
- Obvious anime references
- Duplicate components

The website should remain professional and suitable for both academic presentation and future career opportunities.

---

# 10. Definition of Success

The UI implementation is considered successful when:

- The interface follows UI_GUIDELINES.md.
- The implementation satisfies PRD.md.
- Components are reusable.
- The layout is responsive.
- Information is easy to scan.
- The website feels modern, clean, and professional.
